Description détaillée

Implantation du module graphe_parse.

Le module utilise abondamment des variables statiques car c'est plus pratique que de passer 10 arguments de fonction en fonction...

Les variables sont ré-initialisées au début de parser_graphe, si bien qu'elle sont reliées à cette méthode spécifiquement (il ne s'agit pas d'un état persistant du module, à la fin de la fonction les variables ne servent plus à rien).

enum parser_mode

Documentation des fonctions

◆ parser_graphe()

struct graphe_t* parser_graphe ( const char *  filename)

Fonction qui parse le fichier dont le nom est passé en paramètre. La fonction se charge toute seule d'ouvrir et de fermer le fichier, puis crée un graphe et le peuple avec les informations qui en sont issues (voir introduction du module pour une description de la syntaxe).

En cas d'erreur, la fonction affiche une erreur sur stderr et renvoie NULL.

Le graphe alloué a pour information sur les noeuds (dans le "noeud données") le nom du noeud tel que fourni dans le fichier.

Paramètres
filenamefichier à parser
Renvoie
graphe résultant du parsing, ou NULL si problème
